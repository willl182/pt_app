# Control documental común

**Versión:** 1.0

**Fecha:** 2026-07-14

**Estado:** Aprobable para aplicar durante las fases 3 a 7

## Propósito

Esta carpeta define las reglas editoriales comunes de los nueve entregables.
No sustituye sus contenidos: establece cómo identificarlos, escribirlos,
relacionarlos con evidencia y generar sus formatos de distribución.

## Ruta rápida

1. Consulte `indice_maestro.md` para localizar el documento oficial de E01 a
   E09 y conocer su estado.
2. Copie `plantillas/plantilla_documento.md` al actualizar una fuente.
3. Use `glosario_ciudadano.md` para mantener términos uniformes.
4. Asigne identificadores según `convencion_evidencia.md` y regístrelos en
   `matriz_trazabilidad.csv`.
5. Seleccione el nivel de detalle con `mapa_audiencias.md`.
6. Genere DOCX/PDF según `cadena_generacion.md`.

## Archivos oficiales de esta fase

| Archivo | Función |
|---|---|
| `indice_maestro.md` | Índice contractual y estado de E01–E09 |

| `mapa_audiencias.md` | Separación entre uso, soporte y auditoría |
| `matriz_trazabilidad.csv` | Relación requisito–documento–evidencia |
| `matriz_trazabilidad.md` | Guía legible para interpretar la matriz |
| `cadena_generacion.md` | Flujo controlado Markdown–DOCX–PDF |
| `plantillas/plantilla_documento.md` | Plantilla común reutilizable |
| `plantillas/ejemplo_controlado.md` | Documento mínimo para probar la cadena |

## Limitación contractual

No se encontró en el repositorio el contrato, TDR o acta primaria. Por ello,
los requisitos `REQ-E01` a `REQ-E09` representan el alcance documental del
plan vigente, no una certificación de cobertura contractual. Cuando se reciba
la fuente primaria se crearán requisitos desagregados y se conservará su
referencia exacta.
