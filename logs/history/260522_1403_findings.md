# Technical Finding / Milestone

**Context**: GUI-driven file loading for the PT analysis application

## The Discovery

The application can operate without any preseeded datasets in `data/`, but it
still needs explicit user uploads for configuration inputs that were previously
treated as fixed files.

The key inputs are now:

- `niveles_calaire.csv` for mapping pollutants to generator columns and units.
- `diseno_estabilidad_homogeneidad.csv` for defining the time blocks and source
  columns used by the stability/homogeneity pipeline.

The GUI now provides upload controls for both files, and the preprocessing
script forwards their paths into the pipeline instead of assuming they exist in
`data/metadata/`.

The application no longer relies on `data/processed/*_equipos.csv` as a hidden
fallback for instrumentation data.

## Key Files Affected

- `app.R`
- `scripts/aplicativo/preprocesar_calaire.R`
- `R/preprocessing/pipeline_calaire.R`
- `docs/archivos-carga-gui.md`
