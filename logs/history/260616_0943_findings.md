# Technical Finding / Milestone: Subagent Design & Deliverable Overviews

**Context**: Project Deliverables and Agent Organization (PT Analysis Application)

## The Discovery

We have structured the project's nine deliverables into a coherent development lifecycle and designated nine specialized subagents corresponding to each deliverable. Each subagent has a defined scope:

1.  **`baseliner`**: Manages the static code snapshot integrity (`01_repo_inicial`).
2.  **`api_auditor`**: Scans and documents the function index/signatures (`02_funciones_usadas`).
3.  **`stat_validator`**: Reviews pure mathematical calculations of homogeneity, stability, and robust stats (`03_calculos_pt`).
4.  **`score_auditor`**: Audits performance evaluations and report templates (`04_puntajes`).
5.  **`ui_designer`**: Oversees mockups, wireframes, and navigation schema (`05_prototipo_ui`).
6.  **`shiny_developer`**: Builds interactive reactive Shiny backend and user documentation (`06_app_logica`).
7.  **`viz_engineer`**: Develops charts, dashboards, and plotly heatmaps (`07_dashboards`).
8.  **`release_mgr`**: Handles compilation of final versions and developer guides (`08_beta`).
9.  **`qa_auditor`**: Conducts numerical audits, external feedback resolution, and reproducibility validation (`09_informe_final`).

We also created overview files (`e1.md` through `e9.md`) for all nine folders in `Entregables_pt_app/` linking the files and verification procedures, mapping them to these subagents.
