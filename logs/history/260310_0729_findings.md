# Technical Finding: Implementación dual Opus vs Codex para hallazgos H1-H9

**Context**: Ajustes estadísticos y de usabilidad en CALAIRE-APP (ISO 13528:2022)

## The Discovery

Se implementaron 9 hallazgos del Informe No. 2 en dos ramas paralelas con enfoques distintos:

### Hallazgos estadísticos resueltos

**H1 — Fórmula B.10 (`R/pt_homogeneity.R:86-87`)**
- El código usaba `abs(s_x_bar_sq - sw_sq/m)` que enmascara el caso de ruido dominante
- Fix: `ss_sq <- max(0, s_x_bar_sq - (sw_sq / m))` con `ss <- sqrt(ss_sq)`
- Mismo fix aplicado en `calculate_stability_stats()` y en `funciones_finales.R`

**H2 — MADe de homogeneidad vs participantes**
- El MADe calculado en `pt_homogeneity.R` es de datos de homogeneidad (correcto para criterio)
- Se renombró a `MADe_hom` / `sigma_pt_hom` para evitar confusión con sigma_pt de puntajes
- sigma_pt para puntajes ya se calculaba correctamente de `participant_data$result` en app.R

**H4 — Umbral Algoritmo A**
- Cambiado de `n_part >= 3` a `n_part >= 12`
- ISO 13528:2022 Sec. 9.4 recomienda muestra suficiente para convergencia iterativa
- Para n < 12, usar MADe o nIQR directamente

### Hallazgos de usabilidad resueltos

**H7** — CSV export con `downloadHandler` y encabezados ISO-conformes
**H8** — Zonas de carga con `border-left` coloreado (azul/ámbar/verde) y etiquetas de origen
**H9** — `tableOutput("variance_components")` añadido al UI panel de homogeneidad
