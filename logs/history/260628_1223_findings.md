# Technical Finding / Milestone: Auditoría de documentos formales en testb

**Context**: PT App - documentación formal de entregables E01-E09

## The Discovery

Se auditó el directorio `testb/`, que contiene el paquete documental formal de los entregables del proyecto PT App. El paquete incluye un índice maestro y documentos técnicos para los nueve entregables en formatos Markdown y DOCX.

La auditoría concluyó que el paquete es sustantivo y está correctamente orientado para reemplazar los README como documentos de entrega, pero aún no está listo para entrega final. Se identificaron problemas de reproducibilidad, redacción, trazabilidad de rutas, homogeneidad de plantilla y evidencia de validación.

## Key Results

- Pares `.md`/`.docx`: completos para índice + 9 entregables.
- Integridad DOCX: correcta; `unzip -t` no reportó corrupción.
- Longitud documental: adecuada; los documentos no son resúmenes cortos.
- Estados documentales: mayormente correctos y honestos.
- Tests:
  - E01, E02, E03 y E04 ejecutaron.
  - E07, E08 y E09 ejecutaron correctamente de forma aislada.
  - E05 falló por falta de `library(testthat)` y dependencia incorrecta del directorio de trabajo.
  - E04 pasó con advertencias repetidas por valores asignados faltantes.
  - E07/E08 pasaron con advertencia menor por línea final incompleta en `data/participants_data4.csv`.

## Blocking Finding

`Entregables_pt_app/05_prototipo_ui/tests/test_05_navegacion.R` no es reproducible como está documentado:

- No carga `library(testthat)`.
- Usa `setwd("..")` dentro de pruebas.
- Ejecutado desde raíz falla con `could not find function "test_that"`.
- En ejecución encadenada puede fallar buscando `html/prototipo.html`.

Este hallazgo afecta directamente el documento `testb/documento_tecnico_entregable_05.md`, que afirma que la integridad del entregable se verifica mediante esa suite.

## Key Files Affected

- `testb/auditoria_entregables_testb.md`
- `testb/documento_tecnico_entregable_05.md`
- `Entregables_pt_app/05_prototipo_ui/tests/test_05_navegacion.R`
- `testb/documento_tecnico_entregable_09.md`
- `testb/*.md`

## Recommended Next Action

Corregir E05 primero, porque es el único bloqueo reproducible. Después hacer una pasada editorial sobre `testb/*.md`, normalizar rutas, agregar bitácoras de verificación, crear matriz de evidencia externa para E09 y regenerar DOCX.
