# Problem Record: Preprocesamiento CALAIRE

**Severity**: Medium

## The Problem

### 1. Columnas en minúsculas no reconocidas
Los patrones de coincidencia en `clean_calaire_raw.R` usaban mayúsculas (`^CO_gen$`) pero los datos tenían minúsculas (`co_gen`), causando que no se detectaran los contaminantes.

### 2. Formato de tiempo AM/PM no parseado
Los archivos de ronda 2a usaban formato `01:32:00 PM` que se interpretaba como `01:32:00` (AM) en lugar de `13:32:00`.

### 3. Valores del generador con decimales variables
La ronda 1 tenía valores como `1.41`, `1.411`, `1.412` (32 niveles) en lugar de valores consistentes (5 niveles).

### 4. `mean_value` truncado a 3 cifras significativas
La función `summarise_reference_levels()` usaba `signif(x, 3)` que convertía `121.037` en `121`, perdiendo decimales.

## User-Requested Corrections

1. **Usar valor del generador directamente** como identificador de nivel, sin buscar en tabla de referencia
2. **No redondear valores del generador** - usar el valor exacto
3. **Redondear resultados a 3 decimales** (no 3 cifras significativas)

## The Solution

### 1. Patrones case-insensitive
```r
# Antes: "^CO_gen$"
# Después: "^[Cc][Oo]_?gen$"
```

### 2. Detección de AM/PM
```r
has_ampm <- grepl("[AP]M$", trimws(time_col), ignore.case = TRUE)
fmt_12h_2yr <- "%m/%d/%y %I:%M:%S %p"
```

### 3. Identificación por valor del generador
```r
# Usar valor exacto del generador como label
gen_values <- valid_rows[[gen_col]]
labels <- paste0(gen_values, "-", unit)
```

### 4. Redondeo a 3 decimales
```r
# Antes: round(signif(mean(rows$mean_value), 3), 3)
# Después: round(mean(rows$mean_value), 3)
```

## Learnings to Preserve

- Siempre usar `[Cc]` en patrones regex para soportar minúsculas/mayúsculas
- `signif(x, 3)` ≠ `round(x, 3)` - cifras significativas vs decimales
- Los valores del generador deben usarse tal cual, sin redondeo
- El formato AM/PM requiere patrón `%I:%M:%S %p` y `%p` para AM/PM
