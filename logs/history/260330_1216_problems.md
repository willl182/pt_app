# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Al ejecutar el pipeline completo de validacion GPT53CDX, se obtienen 4446 FAIL de 7665 checks totales (58%), distribuidos en las 5 secciones.

## Root Causes Identificadas

1. **openxlsx2 no instalado**: el script original usaba `openxlsx2` (API `wb_workbook()`, `cx$new()`). Solucion: cambiar a `openxlsx` con API clasica (`createWorkbook()`, `addWorksheet()`, `writeDataTable()`, `saveWorkbook()`).

2. **numpy/pandas no disponibles**: Python script original usaba `import numpy as np; import pandas as pd`. Solucion: reimplementar todo con stdlib (csv, math, collections).

3. **Columnas tipo mixto en bind_rows**: `app_value` era double en unas secciones y character en otras. Solucion: convertir todo a character en `build_master()`.

4. **merge con columna py_value duplicada**: el CSV master ya tenia `py_value` y el join con Python tambien. Solucion: renombrar a `py_value_py` en el join, transferir despues.

5. **Comparacion NA en string_match y py_ok**: varias condiciones `if (!is.na(x))` fallaban con vectores vacios. Solucion: usar `isTRUE()` y checks de longitud.

## Pending Issues

- **4446 FAILs**: sospecha principal es tolerancia 1e-12 demasiado estricta para R vs Python en quantiles y propagacion numerica. Requiere diagnostico comparando valores directamente.

## Learnings to Preserve

- Siempre verificar que paquetes R estan instalados antes de usarlos (openxlsx vs openxlsx2)
- Python stdlib puede reimplementar estadistica basica sin numpy/pandas
- Usar `isTRUE()` para todas las comparaciones booleanas en R donde NA puede aparecer
- Convertir columnas a character antes de `bind_rows()` cuando hay secciones con tipos mixtos
