# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

The O3 validation Excel originally populated `resultado_homogeneidad` and
`resultado_estabilidad` with internal metric dumps. The user clarified that
these sheets must copy the first UI summary tables from `app.R`: "Resumen del
Estudio (Metodo MADe)" and "Resumen del Estudio (Metodo nIQR)".

## User-Requested Corrections

- Homogeneidad and estabilidad must contain the first app tables, not the later
  criteria tables.
- nIQR must live in the same sheet as MADe.
- `algoritmo_A` must include the summary block and iteration count.
- For O3 level 0, Algoritmo A values should be zero.

## The Solution / Workaround

- Replaced the validation rows for `resultado_homogeneidad` and
  `resultado_estabilidad` with `study_summary_tables()`, mirroring the app UI
  values and labels.
- Matched app behavior where stability summary uses `homogeneity_run()` values.
- Updated `algorithm_a_table()` to emit the UI summary fields, including
  initial values, final robust values, winsorized count, total observations,
  iteration count, convergence criterion, tolerance, and first stabilization
  iteration.

## Learnings to Preserve

For this validation workbook, "copy paste literal" means mirror the visible
`app.R` UI tables and labels, not the underlying calculation objects.
