# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

The Algorithm A iteration table in `app.R` broke after convergence-criterion changes. The render path attempted to access a convergence column that was missing in some result objects, which triggered:

`Warning: Error in $<-.data.frame: replacement has 0 rows, data has 2`

This happened inside `output$algoA_iterations_table`.

## User-Requested Corrections

- Keep Algorithm A convergence at the 3rd significant figure.
- Keep numeric display at 4 significant figures.
- Do not break the Algoritmo A iteration table.

## The Solution / Workaround

- Restored the Algorithm A stopping rule in `ptcalc/R/pt_robust_stats.R` to `signif3`.
- Left `format_num()` unchanged so the UI still shows 4 significant figures.
- Made `output$algoA_iterations_table` tolerant to:
  - `signif3_converged`
  - `signif4_converged`
  - a `delta_max < tolerance` fallback

## Learnings to Preserve

- Do not couple the table renderer to a single convergence-column name when cached or older results may still be present.
- The convergence rule and the display precision are separate concerns and should be handled independently.
- When changing Algorithm A internals, test both the raw function return and the Shiny render path.
