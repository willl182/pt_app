# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Two user-facing issues were resolved in this session:

1. `grubbs_summary` raised `Error in data.frame: arguments imply differing number of rows: 1, 0` when running the Grubbs summary for `summary_n13.csv`.
2. Algorithm A still exposed long decimal outputs in the UI and CSV export even after the workflow had been intended to be rounded/truncated.

## User-Requested Corrections

- Fix the Grubbs summary error.
- Keep Algorithm A output visually truncated/rounded.
- Increase the presentation rule from 3 significant figures to 4 significant figures.
- Apply the same formatting to the CSV export.

## The Solution / Workaround

- Hardened `grubbs_summary` with empty-combination guards and scalar-safe row assembly.
- Replaced hard-coded `formatRound(..., digits = 9)` table formatting in Algorithm A outputs with `format_num()`.
- Changed `format_num()` default precision from 3 to 4 significant figures.
- Updated the Algorithm A CSV export to format numeric fields before writing.

## Learnings to Preserve

- If a Shiny reactive error says `data.frame` row counts differ, inspect row assembly, not just the source data.
- Centralize display precision in one formatter when multiple UI and export paths need the same rule.
- Avoid hard-coded `formatRound()` settings when the app already has a canonical numeric formatter.
