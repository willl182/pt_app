# Technical Finding / Milestone

**Context**: CALAIRE preprocessing architecture

## The Discovery

The CALAIRE preprocessing pipeline should live inside `pt_app` as a project-specific internal module. It should not be placed in `ptcalc`, because `ptcalc` is intended to remain a generic statistical calculation layer. It also should not initially be implemented in `calaire-app`, because the preprocessing is metrological/statistical and better handled offline in R before web integration.

A git submodule is not recommended at this stage. The pipeline belongs to the `pt_app` business flow and can be organized as an internal module under `R/preprocessing/` with an orchestrator script under `scripts/`.
