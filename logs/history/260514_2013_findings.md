# Technical Finding / Milestone

**Context**: Consolidated participant results and uncertainty loading in the Shiny app

## The Discovery

`data/processed/ronda_1_completa.csv` already contains the data needed by both previous inputs:

- Participant summary fields: `participant_id`, `pollutant`, `level`, `mean_value`, `sd_value`.
- Participant uncertainty field: `u_value`, now normalized in `app.R` as `u_i`.
- Reference rows can be identified by `tipo == "referencia"` and normalized as `participant_id = "ref"`.

This means the separate "Incertidumbre de participantes" upload is unnecessary when using the consolidated pipeline output.

## Implementation

- `auto_load_participants()` now prefers `*_completa.csv` files from `data/processed/`.
- Manual summary uploads normalize `instrument -> participant_id`, `tipo == "referencia" -> participant_id = "ref"`, and `u_value -> u_i`.
- `pt_data_df()` no longer reads from `input$pt_data_file`; it first uses uncertainty embedded in the loaded summary data.
- The UI upload grid no longer displays the separate uncertainty input.
- The consolidated participants input text now states that it includes participant results and uncertainty.

## Verification

- `app.R` parses successfully.
- A direct check confirmed that `data/processed/ronda_1_completa.csv` satisfies both the summary and uncertainty column contracts after normalization.
- Full `testthat::test_dir("tests/testthat")` still fails because unrelated documentation fixtures are missing: `tools/inventory_docs.R` and `final_docs/*.md`.
