# Technical Finding / Milestone

**Context**: PT App Validation - validation_1 semiautomatic rerun

## The Discovery

Two concrete issues blocked the rerun of `validation_1` with custom inputs:

- The R wrapper was sourcing all stage scripts before executing any of them.
  Because each script defines shared globals such as `OUTPUT_R_CSV`, later
  sources overwrote earlier stage constants. This caused stage 1 to write to
  the stage 5 output path.
- The stage 1 R comparison branch still used a fragile scalar `if (...)`
  expression when building the comparison table. When a missing value reached
  that branch, the run aborted with `missing value where TRUE/FALSE needed`.

## The Fix

- Changed `scripts/run_validation_semiauto.R` to source and execute each stage
  sequentially instead of sourcing every stage up front.
- Patched `validation_1/stage_01_robust_stats.R` to compute the finite-value
  check once and reuse it safely for both difference columns.

## Result

- The semiautomatic R wrapper now completes end-to-end on the requested inputs.
- The Python wrapper also completes end-to-end on the same inputs.
- Stage 5 finishes with `1152 PASS / 0 FAIL`.

## Key Files Affected

- `scripts/run_validation_semiauto.R`
- `validation_1/stage_01_robust_stats.R`
- `logs/CURRENT_SESSION.md`
