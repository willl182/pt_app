# Problem Record: Informe final, graficos y heatmap

**Severity**: Medium

## The Problem

El informe final tenia dos problemas reportados por usuario: graficos ausentes o
mal renderizados, y un heatmap que no correspondia con la seleccion esperada.
Ademas, una seleccion de un solo participante estaba generando resultados para
todos en Anexo C.

## User-Requested Corrections

- Trabajar sobre `reports/report_template.Rmd`.
- Corregir los graficos del informe.
- Evitar que Anexo C genere todos los participantes cuando se selecciona uno.
- Verificar que el heatmap corresponda con la seleccion.

## The Solution / Workaround

- Se agrego `library(patchwork)` al setup del Rmd para imprimir correctamente
  objetos compuestos de Anexo C.
- Se agrego selector `report_participant` en `app.R` y el filtro se aplica en
  `report_participant_data()`.
- Se corrigio `report_heatmaps()` para que no use `report_participant`; el
  heatmap representa todos los participantes del `n_lab`, nivel, metodo y
  metrica seleccionados.
- Se agrego subtitulo al heatmap con `n`, nivel, metodo e indicador.
- Se valido un render minimo a Word y el `.docx` contiene imagenes embebidas.

## Learnings to Preserve

- El selector de participante es solo para Anexo C. No reutilizarlo para las
  secciones generales del informe.
- Los graficos `patchwork` pasados como parametros a R Markdown requieren que
  la plantilla cargue `patchwork` antes de imprimirlos.
- Validar graficos de Word inspeccionando `word/media` dentro del `.docx`.
