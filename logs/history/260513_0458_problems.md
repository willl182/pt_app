# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

After changing several DT tables to use `format_numeric_columns()`, Shiny threw:

`no applicable method for 'mutate' applied to an object of class "c('datatables', 'htmlwidget')"`

The helper was being called on `DT::datatable()` widgets, but it only handled plain data frames.

## User-Requested Corrections

- Restore the app so it runs without the DT `mutate()` error.
- Keep the 4 significant figure display rule.
- Avoid reintroducing excessive decimal precision.

## The Solution / Workaround

- Extended `format_numeric_columns()` to detect `htmlwidget` inputs.
- For widgets, it now calls `DT::formatRound()` directly.
- For plain data frames, it keeps using `mutate(across(..., ~ format_num(.x)))`.

## Learnings to Preserve

- Do not reuse a data-frame formatting helper on `DT` widgets unless it branches by class.
- Presentation helpers in Shiny should support both pre-widget and post-widget formatting paths when the codebase mixes `renderTable()` and `renderDataTable()`.
- When correcting numeric precision, verify the output type before swapping in a shared helper.
