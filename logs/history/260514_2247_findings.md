# Technical Finding / Milestone

**Context**: `app.R` UI and PT score/outlier refinements

## The Discovery

The Grubbs outlier workflow was clearer when the tested values are the individual hourly means (`mean_h1`, `mean_h2`, `mean_h3`) rather than only the participant-level consolidated `mean_value`. The consolidated `mean_value` still helps as a visual reference, so it is shown as dashed vertical lines in the outlier histogram.

For Puntajes EA, `sigma_pt` is important context for interpreting z and z' and should be visible in the main score tables and metric-specific tables. Score equations should be rendered using MathJax/LaTeX rather than plain text.

The participant detail view should render each chart as an independent Plotly output. A single `plotly::subplot()` with four panels can compress or overlap charts in Shiny.
