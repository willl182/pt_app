# Implementacion Fase 2 completada — Homogeneidad

**Context**: Validacion downstream de Algoritmo A — Etapa 2 homogeneidad

## Lo que se implemento

### R (`validation/stage_02_homogeneity.R`)
- Carga de datos desde `homogeneity_n13.csv` con `load_wide_data()`
- Pivoteo de réplicas a formato ancho (sample_1, sample_2)
- Cálculo de 13 métricas de homogeneidad:
  - g, m, general_mean_homog, x_pt, s_x_bar_sq, sw, ss_sq, ss
  - sigma_pt, MADe, u_sigma_pt, criterio_c, criterio_expandido
- Fórmula ss_sq: `abs(s_x_bar_sq - sw_sq/m)` (usa abs() como ptcalc)
- Criterio expandido: versión 3 args con tabla F1/F2 (g clamped a [7,20])
- Output: `stage_02_homogeneity_r.csv` (intermedio)

### Python (`validation/stage_02_homogeneity.py`)
- Carga idéntica a R con `load_wide_as_matrix()`
- Implementación de `variance()` (ddof=1), `median()`, `quantile_type7()`
- Cálculo de las mismas 13 métricas con fórmulas idénticas
- Tabla F1/F2 hardcodeada para criterio expandido
- Comparación tripartita leyendo CSV intermedio de R
- Generación de `stage_02_homogeneity.csv` (tabla canónica)
- Generación de `stage_02_homogeneity_report.md`

## Observaciones

- **m=2 para todos los combos**: se usa fórmula de rangos para sw
- **g=13 para todos los combos**: sin edge cases (sigma_pt > 0 en todos)
- **ss_sq usa abs()**: en lugar de clamp a 0 (versión ptcalc)
- **Criterio expandido**: usa F1=1.75, F2=0.80 (g=13 clamped a 13)
- **O3 0 nmol/mol**: sigma_pt muy pequeño (6.4e-6) pero finito, no edge case

## Flujo de ejecución

```bash
Rscript validation/stage_02_homogeneity.R     # Paso 1: genera CSV intermedio R
python3 validation/stage_02_homogeneity.py     # Paso 2: genera comparación + reporte
```
