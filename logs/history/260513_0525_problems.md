# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

The validation report for O3 was not formatted the way the user wanted. Each stage table listed R, Python, and App, but the Excel result was emitted as a separate section instead of being included as another row in the same comparison table.

## User-Requested Corrections

- Put the Excel result row inside each stage table.
- Do not keep Excel results in a separate “Resultados de Excel” section.

## The Solution / Workaround

- Updated `validation_1/validation/informe_validacion_o3.md`.
- Added an `Excel` row to each stage table in sections 3 through 7.
- Removed the standalone Excel results section.

## Learnings to Preserve

- For validation artifacts, Excel evidence should be shown inline with the stage comparisons when the goal is direct cross-checking.
- A separate appendix is not equivalent to a stage-level comparison table when the user explicitly asks for per-stage Excel rows.
