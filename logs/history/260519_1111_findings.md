# Technical Finding / Milestone

**Context**: Guía del procedimiento de ronda para Calaire y participantes

## The Discovery

Se elaboró un plan detallado para documentar el procedimiento completo de una ronda de ensayo de aptitud, separando claramente:

1. **Procedimiento del participante**: carga de resultados resumidos en aplicativo, sin usar scripts, sin diligenciar campos estructurales (ronda, contaminante, corrida, nivel, unidad).

2. **Procedimiento de Calaire**: basado en scripts internos del repositorio (preprocesar_calaire.R, run_preprocessor_calaire.R, unir_rondas.R, etc.).

## Campos que carga el participante (pantalla del aplicativo)

| Campo visible | Significado técnico |
|--------------|---------------------|
| Dato 1 | Primer dato/promedio |
| Dato 2 | Segundo dato/promedio |
| Dato 3 | Tercer dato/promedio |

| u(x) | u_xi (incertidumbre estándar de xi) |
| u(x) exp | U_xi (incertidumbre expandida de xi) |

## Brecha técnica identificada

El CSV actual `ronda_1_completa.csv` no almacena explícitamente `U_xi` ni `k`. Calaire necesitará reportar `k` posteriormente, calculable como `k = U_xi / u_xi`. La corrección del aplicativo/consolidado para almacenar ambos valores queda para fase posterior.
