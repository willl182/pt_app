# Problem Record (Troubleshooting)

**Severity**: Low

## The Problem

Varias salidas renderizadas de la app mostraban notación técnica como texto plano o nombres internos:

- El operador menor o igual se veía como `<=` o podía interpretarse visualmente como `< =`.
- `u_sigma_pt` aparecía como etiqueta visible, exponiendo el nombre interno en vez de la notación matemática.
- La nota de compatibilidad metrológica usaba texto plano para `D_2a`, `D_2b` y `x_pt`, sin subíndices.

## User-Requested Corrections

- Renderizar menor o igual como símbolo `≤`, no como `<=`.
- Renderizar `u_sigma_pt` como incertidumbre de sigma: `u(σ_pt)`.
- Renderizar la nota como: `D_2a = x_pt(Ref) - x_pt(2a); D_2b = x_pt(Ref) - x_pt(2b)`, con subíndices en la UI.

## The Solution / Workaround

Se editó `app.R` solo en la capa de presentación:

- Las cadenas de conclusiones de homogeneidad y estabilidad usan `≤`.
- La conclusión de prueba t usa `p ≤ 0.05`.
- La evaluación de `u(x_pt)_def` usa `≤`.
- Las tablas de resumen cambian la etiqueta visible de `u_sigma_pt` a `u(σ_pt)`.
- La nota de compatibilidad metrológica usa `HTML()` con `<sub>` para subíndices.

La sintaxis fue validada con:

```r
Rscript -e 'invisible(parse("app.R")); cat("parse OK\n")'
```

## Learnings to Preserve

- En esta app, varias conclusiones se reutilizan tanto en `renderUI(HTML(...))` como en salidas de texto/reportes. Para operadores matemáticos simples, preferir el símbolo Unicode `≤` sobre entidades HTML como `&le;`.
- No renombrar variables internas como `u_sigma_pt` si solo se corrige renderizado; cambiar únicamente etiquetas visibles.
- Para notación con subíndices en UI Shiny, usar `HTML()` con etiquetas `<sub>` en textos estáticos.
