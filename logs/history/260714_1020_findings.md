# Technical Finding / Milestone

**Context**: Planificación de documentación contractual del aplicativo PT

## The Discovery

El paquete `Entregables_pt_app/` contiene nueve entregables contractuales, pero
varios documentos describen hitos históricos (`v06`, `v07`, beta o prototipo)
y no el comportamiento completo de `app.R` vigente. La documentación final
debe distinguir explícitamente evolución histórica, operación actual y
evidencia de validación.

Existe un flujo Playwright en `dgpsea03/scripts/tomar_capturas.js` con 17
capturas de las pantallas principales. Es una base reutilizable, pero para una
auditoría contractual debe regenerarse desde la versión final, ampliarse con
variantes de puntajes, errores de validación y resolución menor, e incluir
fecha, commit, datos de prueba, índice y hashes.

Se definió un plan de siete fases que cubre E01-E09, lenguaje para público no
técnico, anexos técnicos, control documental, trazabilidad y una matriz mínima
de 19 escenarios visuales.
