# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Stage 5 was reporting `960 FAIL` because it was counting `_eval` rows as if they were numeric comparisons. That polluted the validation summary and made the report look broken even though the numeric scores were fine.

## User-Requested Corrections

- Remove the comparison of `_eval` fields.
- Keep only the numeric validation summary.

## The Solution / Workaround

- Updated `validation_1/stage_05_scores.py` to compare only numeric metrics.
- Rounded numeric values to 4 decimals before comparing, so tiny floating-point noise does not create false failures.
- Updated `validation_1/stage_05_scores.R` so `_eval` values stay as text in the exported data instead of being coerced to `NA`.
- Regenerated the final report with `validation_1/generate_informe_validacion_o3.R`.

## Learnings to Preserve

- Do not mix categorical `_eval` labels with the numeric PASS/FAIL summary.
- Validation summaries should report only the metrics the delivery actually cares about.
- Small floating-point differences should be normalized before comparison when the report is meant to reflect practical equivalence.
