# Problem Record (Troubleshooting)

**Severity**: High

## The Problem

La app mezclaba conceptos metrologicos distintos y la UI podia inducir
interpretaciones erroneas:

- La tabla de referencia decia `u(xpt) / sd declarada`, aunque el usuario
  quiere tratar ese dato como `u(xpt)`.
- Compatibilidad metrologica fallaba con:
  `Error in left_join: Join columns in y must be present in the data`.
- `Referencia (1)` y `Expertos (4)` aparecian con los mismos parametros,
  aunque `Expertos (4)` debe usar su formula de `sigma_pt`.
- Las tablas no dejaban claro si `u(x_pt)` era base o si ya incluia
  homogeneidad y estabilidad.

## User-Requested Corrections

- Cambiar la etiqueta de referencia a `u(xpt)` y no mencionar
  `sd declarada`.
- Arreglar el error de compatibilidad metrologica cuando hay grupos con
  menos de dos datos.
- Separar `Referencia (1)` de `Expertos (4)`: pueden tener el mismo `x_pt`,
  pero no la misma `sigma_pt`.
- Para `Expertos (4)`, usar `u(x_pt) = 0.003 * x_pt`.
- Mostrar claramente en la app `u(x_pt) base`, `u(x_pt)_def` y
  `U(x_pt)_def`.

## The Solution / Workaround

- En `output$reference_table`, la columna se renombro a `u(xpt)`.
- En `metrological_compatibility_data()`, se agregaron helpers locales para
  construir tibbles vacios tipados cuando consenso o Algoritmo A no tienen
  filas disponibles. Esto conserva las columnas de join requeridas.
- Se agrego `calculate_expert_u_xpt(x_pt)` con la regla
  `0.003 * x_pt`.
- En las rutas de seleccion, compatibilidad, resumen global, reporte y
  exportacion:
  - `Referencia (1)` usa la incertidumbre de referencia como base.
  - `Expertos (4)` usa `sigma_pt = a * x_pt + b` y
    `u(x_pt) base = 0.003 * x_pt`.
- Las tablas de resumen, vista general, metricas individuales y detalle por
  participante ahora exponen:
  - `u(x_pt) base`
  - `u(x_pt)_def`
  - `U(x_pt)_def`

## Learnings to Preserve

- No etiquetar una misma columna como `u(xpt)` y `sd declarada`; en esta app
  se debe presentar como `u(xpt)`.
- No asumir que `Referencia (1)` y `Expertos (4)` comparten `sigma_pt`.
- Mantener separados los conceptos:
  - `u(x_pt) base`: incertidumbre propia del metodo.
  - `u(x_pt)_def`: incertidumbre combinada con homogeneidad y estabilidad.
  - `U(x_pt)_def`: incertidumbre expandida usada en En.
- En joins reactivos, cuando una rama puede devolver cero filas, devolver un
  tibble vacio con el esquema completo evita errores de `left_join()`.
