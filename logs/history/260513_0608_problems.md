# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

The final validation report was too terse and inconsistent:

- it lacked formulas
- it mixed method validation with a single-method reading
- it showed raw precision and scientific notation in places
- it was being maintained manually instead of from a script

## User-Requested Corrections

- clean the text
- show formulas in the report
- make Stage 4 about value assignment, not uncertainty chain wording
- show scores by method
- force all visible numbers to 4 decimals
- generate the final report by script

## The Solution / Workaround

- rewrote the final report structure to include formulas for Stage 4 and Stage 5
- created `validation_1/generate_informe_validacion_o3.R`
- regenerated the final report from the canonical stage outputs
- normalized Stage 4 and Stage 5 outputs to 4-decimal display

## Learnings to Preserve

- the final report should be derived from canonical stage artifacts, not edited by hand
- method-specific validation must remain explicit in both Stage 4 and Stage 5
- any numeric text in validation artifacts should be normalized to 4 decimals before release
