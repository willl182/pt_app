# Problem Record (Troubleshooting)

**Severity**: Medium

## The Problem

Several UI changes initially surfaced information in a way that was technically present but not useful to the user:

- The reference tab showed sections for separated CALAIRE reference and hourly CALAIRE reference even when not activated or unavailable.
- Score equations were displayed as plain text instead of rendered mathematical notation.
- Participant detail charts were grouped in a single Plotly subplot, causing overlap/compression.

## User-Requested Corrections

- Remove the visible UI blocks for `Referencia CALAIRE separada` and `Referencia CALAIRE horaria`.
- Render equations properly, not as plain text.
- Separate the four participant-view charts so they do not overlap.

## The Solution / Workaround

- The reference tab now shows only `Resultados de Referencia`.
- A helper `score_equation()` wraps LaTeX strings with `withMathJax()`.
- Participant charts now use four independent outputs:
  - values/reference
  - z
  - zeta
  - En

## Learnings to Preserve

- Do not expose diagnostic/backend reference tables by default unless the user explicitly asks for them.
- When adding formulas to Shiny UI, use MathJax immediately.
- Avoid multi-chart Plotly subplots in constrained Shiny tabs when users need readable individual charts.
