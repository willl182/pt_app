# Technical Finding: Preprocesamiento CALAIRE

**Context**: PT Analysis App - Procesamiento de datos de rondas de calibración

## The Discovery

### 1. Detección automática de separadores
El lector `read_calaire_raw.R` ahora detecta automáticamente:
- Punto y coma (`;`)
- Coma (`,`)
- Tab (`\t`)
- Espacios en blanco (`\\s+`)

Esto permite leer archivos con diferentes formatos de exportación.

### 2. Soporte para formato AM/PM
El parser de timestamps en `clean_calaire_raw.R` ahora detecta y parsea correctamente:
- Formato 24h: `04/23/26 14:30`
- Formato 12h AM/PM: `04/30/26 01:32:00 PM` → `13:32:00`

### 3. Identificación de niveles por valor del generador
En lugar de buscar coincidencias en una tabla de niveles fijos, el sistema ahora:
- Usa el valor exacto del generador como identificador del nivel
- Agrupa datos por cambios en el valor del generador
- No requiere tolerancias ni coincidencias exactas con nominales predefinidos

### 4. Redondeo consistente a 3 decimales
Todos los valores numéricos de salida se redondean a 3 decimales:
- `mean_h1`, `mean_h2`, `mean_h3` (promedios horarios)
- `mean_value` (promedio de promedios horarios)
- `sd_value` (desviación estándar)
- `u_value` (incertidumbre)
- `generated_mean`, `generated_sd` (estadísticas del generador)

## Key Files Affected

- `R/preprocessing/read_calaire_raw.R` - Detección de separadores
- `R/preprocessing/clean_calaire_raw.R` - Parsing AM/PM, normalización de columnas
- `R/preprocessing/hourly_averages.R` - Cálculo de promedios con redondeo
- `R/preprocessing/pipeline_calaire.R` - Pipeline principal
- `scripts/run_preprocessor_calaire.R` - Script de ejecución
- `scripts/unir_ronda_1.R` - Unión de archivos ronda 1
- `scripts/unir_ronda_2.R` - Unión de archivos ronda 2
