# Technical Finding / Milestone

**Context**: Bootstrap reproducible para homogeneidad y estabilidad.

## The Discovery

El flujo correcto para auditar los datos simulados requiere tres niveles de
salida:

1. Datos minutales simulados: `data/processed/bootstrap_minute_simulations.csv`.
2. Datasets horarios simulados: `data/processed/bootstrap_hourly_datasets.csv`.
3. Archivos finales en formato del aplicativo:
   `data/processed/ronda_1_homogeneidad.csv` y
   `data/processed/ronda_1_estabilidad.csv`.

El archivo horario es la fuente directa de los finales y permite verificar
facilmente que cada valor de homogeneidad/estabilidad corresponde a un
`dataset_id` especifico.

## Implementation Notes

- El script nuevo exige `--input`; no usa `datos_ronda_1_r.csv` como default.
- `--datasets` controla cuantas simulaciones se guardan. Se uso 200 para mayor
  estabilidad del conjunto intermedio.
- La semilla usada fue `13528`.
- El registro `data/metadata/bootstrap_homogeneity_stability_register.csv`
  documenta `study_type`, `replicate`, `sample_id`, `dataset_id`, `seed`,
  `input_path` y `minute_pool_n`.
