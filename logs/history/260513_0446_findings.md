# Technical Finding / Milestone

**Context**: PT app presentation and outlier summary fixes

## The Discovery

The Grubbs error was not caused by differences between `summary_n13.csv` and `summary_n13a.csv`. The real failure was fragile row construction inside `grubbs_summary`, where an edge case could feed a zero-length value into `data.frame()`.

For Algorithm A, the long decimal output was coming from presentation code, not the estimator itself. The cleanest fix was to centralize display formatting through `format_num()` and use it consistently in UI and CSV export.
