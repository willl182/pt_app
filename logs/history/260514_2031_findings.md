# Technical Finding / Milestone

**Context**: Reference selection rule for consolidated PT data loading

## The Discovery

The UI wording around "Referencia CALAIRE procesada" was ambiguous. The intended behavior is:

- If the reference checkbox is off, the reference must come from the consolidated participants file as `participant_id = "ref"`.
- If the reference checkbox is on, the user must upload an individual CALAIRE reference CSV.

The app should not silently force or auto-load a separate reference file when the checkbox is off.

## Implementation

- `calaire_reference_df()` now returns `NULL` unless `input$use_calaire_reference` is active.
- When active, `calaire_reference_df()` reads only `input$calaire_reference_file`.
- The UI now shows a reference file upload only inside a `conditionalPanel()` when the checkbox is active.
- If the checkbox is active and no CSV is uploaded, `pt_prep_data()` raises a clear validation error.
- UI copy was updated to explain that the default reference comes from the consolidated file as `participant_id='ref'`.
