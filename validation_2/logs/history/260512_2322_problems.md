# Problem Record: Scripts viejos de validación con referencias rotas

**Severity**: Medium

## The Problem

Los scripts de Etapa 2 en `validation/` (directorio viejo) referenciaban `source("validation/helpers.R")` — un archivo inexistente. Además, el script Python `stage_02_homogeneity.py` antiguo usaba `canonical_row` que no existe en `validation_2/helpers.py`.

## User-Requested Corrections

"Implementar la fase 2 del plan desde cero. OMITIR lo que se hizo anteriormente."

Se crearon archivos completamente nuevos en `validation_2/` sin depender de nada en `validation/`.

## The Solution / Workaround

Se crearon `validation_2/stage_02_homogeneity.R` y `validation_2/stage_02_homogeneity.py` desde cero con:

1. `source("validation_2/helpers.R")` (ruta correcta)
2. Funciones de cálculo directas sin depender de `canonical_row`
3. Output a `validation_2/outputs/`
4. Comparación R vs Python automática en el script Python
5. Verificación contra ptcalc en el script R

## Learnings to Preserve

1. **Nunca referenciar `validation/`** — trabajar exclusivamente en `validation_2/`
2. **El helper Python `canonical_row` no existe** — usar diccionarios directos
3. **`load_wide_data` retorna un DataFrame con `sample_id`** — filtrar con `c.startswith("sample_") and c.replace("sample_", "").isdigit()`
4. **Siempre verificar contra ptcalc** como fuente de verdad para R
5. **Documentar discrepancias conocidas** (ej: criterion_expanded con 2 vs 3 args)
